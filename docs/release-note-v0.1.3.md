## fooks v0.1.3

fooks v0.1.3 is a small follow-up release focused on making setup feel production-ready instead of debug-oriented, without widening the documented support boundary.

### What's new

- Improved setup UX so readiness messaging feels ready rather than debuggy.

### Release boundary reminders

- `fooks setup` remains an explicit activation step; package installation alone should not be described as automatic hook mutation.
- Codex repeated-file acceleration remains the primary supported path.
- Claude and opencode remain narrower helper/setup surfaces in this release slice.

### Still not claimed

- Stable runtime-token or latency wins.
- Provider billing-token, invoice, or charged-cost proof.
- New environment support beyond the existing documented release boundary.
