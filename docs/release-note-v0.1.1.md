## fooks v0.1.1

fooks v0.1.1 keeps the public claim surface conservative while preparing the package for public distribution under the npm-safe package name. The installed CLI remains `fooks`; the package identity remains `fxxk-frontend-hooks`.

### What's new

- Published the package under the npm-allowed distribution name.
- Removed private planning drafts from the public tree before release.
- Added design-review metadata support behind an explicit opt-in instead of widening the default public surface.
- Hardened release-smoke claim checks around provider-cost and billing-token wording.
- Clarified README and benchmark wording so release-facing claims stay evidence-bounded.
- Opened the guarded Codex TS/JS same-file beta path without changing the broader release boundary.

### Release boundary reminders

- `fxxk-frontend-hooks` is the npm package name; `fooks` is the CLI command.
- Codex remains the automatic repeated-file hook path through `fooks setup`.
- Claude remains scoped to project-local context hooks for `SessionStart` / `UserPromptSubmit`.
- opencode remains a narrower prepared tool/slash-command path, not a full Codex-equivalent automatic interception path.

### Still not claimed

- Stable runtime-token or latency wins across tasks.
- Provider billing-token, invoice, dashboard, or charged-cost proof.
- Broad React Native or WebView support beyond explicitly gated evidence.
- Whole-repo compression or universal file-read interception.
