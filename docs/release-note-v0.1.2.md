## fooks v0.1.2

fooks v0.1.2 is a packaging-identity follow-up release. The functional claim surface stays the same as v0.1.1; this release mainly aligns the npm-safe package identity wording with what users should actually install.

### What's new

- Switched to the requested npm-safe `fxxk-frontend-hooks` package identity.

### Release boundary reminders

- Install examples should use `npm install -g fxxk-frontend-hooks`.
- The installed CLI command remains `fooks`.
- This release does not expand runtime guarantees, provider-cost claims, or supported frontend domains.

### Still not claimed

- Stable runtime-token or time savings.
- Provider billing-token or charged-cost proof.
- Broad RN/WebView support based on `.tsx` parsing alone.
